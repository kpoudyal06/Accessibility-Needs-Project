# Accessibility Needs Project Frontend Overview

## File Overview

### 1. `app.R`
The entry point of the application. It handles the global configuration and environment setup.
* **Initialization:** Increases the maximum upload size to **30MB**.
* **Security:** Retrieves sensitive credentials (system user, cluster credentials) from environment variables.
* **Helper Functions:** Contains `sync_and_clean_pdf`, a utility function that uses WinSCP to securely transfer files from the local machine to the UMBC cluster and deletes the local temporary copy immediately after.
* **Execution:** Sources `ui.R` and `server.R` to launch the app.

### 2. `ui.R`
Defines the User Interface and visual layout of the application using `bslib`.
* **Navigation:** Uses a fixed-top navbar with four primary sections:
    1.  **Upload File:** A dynamic zone for file submission and PDF previewing.
    2.  **Track Progress:** A simple interface to check the status of a specific Job ID.
    3.  **Retrieve File:** A dedicated page for downloading finished individual files and providing user feedback.
    4.  **Job Actions:** A technical dashboard to fetch real-time logs (`.out` files) from the cluster and download the final processed `.zip` packages.

### 3. `server.R`
Contains the "brain" of the application, managing all logic and data processing.
* **Dynamic UI:** Controls the transition between the upload form and the "Success" screen.
* **Job Generation:** Creates unique IDs using the format `JOB_YYYYMMDD_HHMMSS_RANDOM`.
* **Cluster Integration:** Generates dynamic WinSCP scripts on-the-fly to create remote directories, upload files, and execute shell commands (like `cat`) on the cluster.
* **Download Logic:** Bundles processed `.pdf` and `.html` files from the cluster into a `.zip` archive for the user.

## Notable Locations & Paths

### Local (WSL/Windows) Paths
The app operates across the Windows Subsystem for Linux (WSL) and the host Windows filesystem.

| Item | Path (WSL / Windows) |
| :--- | :--- |
| **WinSCP Executable** | `/mnt/c/Program Files (x86)/WinSCP/WinSCP.com` |
| **Project Root** | `C:\Users\[USER]\Downloads\Accessibility-Needs-Project\frontend` |
| **Temporary Uploads** | `...\frontend\websitePDFs\[JOB_ID]\` |
| **Download Staging** | `...\frontend\downloads\[JOB_ID]\` |
| **PDF Previews** | A temporary `www` directory generated by `tempdir()` |

### Remote (Cluster) Paths
Files are moved to the UMBC cluster for remediation processing.

| Item | Path |
| :--- | :--- |
| **Upload Destination** | `/umbc/class/cmsc447sp26/common/Accessibility-Needs-Project/backend/fileUploadLocation/[JOB_ID]/` |
| **Output Location** | `.../[JOB_ID]/output/` (Contains the processed `.pdf` and `.html` files) |
| **Log Files** | `.../[JOB_ID]/*.out` |

## Key Workflows

1.  **Uploading:** When a user submits a PDF, it is renamed with their Student ID, placed in a unique job folder locally, and then pushed to the cluster via WinSCP. The local folder is deleted immediately after the transfer.
2.  **Tracking:** Users enter their Job ID. The app connects to the cluster and runs `cat` on the `.out` files to show the user the current processing logs.
3.  **Retrieval:** The app searches the cluster's `output` folder for the specific Job ID. It pulls all `.pdf` and `.html` files back to a local staging area, zips them up, and serves them to the user's browser.

> **Note:** This application requires a configured `.Renviron` file containing `WIN_USER`, `CLUSTER_USER`, and `CLUSTER_PWD` to function correctly.
